# OdtThumbnailHandler

[![License: GPL v3](https://img.shields.io/badge/License-GPLv3-blue.svg)](https://www.gnu.org/licenses/gpl-3.0)

An elegant, lightweight, and high-performance thumbnail handler for **MediaWiki (1.40+)** to automatically generate preview images for OpenDocument files (`.odt`, `.ods`, `.odp`).

---

## 🇬🇧 English Documentation

### ✨ Key Features

* 🚀 **Zero Overhead:** Does **not** require LibreOffice, OpenOffice, or heavy external tools (like `unoconv`) installed on the server. Instead, it extracts the pre-rendered PNG preview (`Thumbnails/thumbnail.png`) that office suites natively embed inside the ODF zip archive upon saving.
* 🧠 **RAM-only Processing:** Image resizing happens entirely in RAM using PHP's GD library. No heavy system processes are spawned.
* 🛡️ **`FileBackend` Native Compatibility:** Correctly interfaces with MediaWiki's storage abstraction layer (`mwstore://`), working flawlessly on both local setups and distributed cloud environments (e.g., AWS S3).
* 🔄 **Intelligent Fallback:** If an ODF file is generated by automated third-party software/scripts and lacks an internal preview, the extension automatically loads a format-specific fallback icon, preventing `0x0` database errors or page crashes.

### 📋 Requirements

* **MediaWiki:** 1.40 or higher (Tested on **MediaWiki 1.43.x**)
* **PHP:** 8.0 or higher
* **PHP Extensions:** `php-zip` and `php-gd`

### 🛠️ Directory Structure

Place the extension inside your `extensions/` directory with the following structure:

```text
OdtThumbnailHandler/
├── extension.json
├── src/
│   └── OdtHandler.php
└── resources/
    ├── default-odt.png
